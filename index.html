ข้อมูลที่แอปใช้

แอปอาจใช้หรือจัดเก็บข้อมูลต่อไปนี้เพื่อให้ผู้ใช้สามารถตั้งค่าและควบคุมอุปกรณ์ได้:

ประเภทข้อมูล ตัวอย่าง แหล่งที่มา การจัดเก็บ
ข้อมูล Device Group ชื่อกลุ่ม, รูปแบบสีของกลุ่ม ผู้ใช้สร้างหรือแก้ไขเองในแอป เก็บใน SQLite บนอุปกรณ์มือถือ
ข้อมูลอุปกรณ์ IoT ชื่ออุปกรณ์, MAC address, ประเภทอุปกรณ์, สถานะ setup, IP address, port มาจากการค้นหาอุปกรณ์ผ่าน UDP discovery และ device profile ข้อมูลอุปกรณ์ที่ setup แล้วถูกเก็บใน SQLite
ข้อมูลการควบคุมอุปกรณ์ สถานะ power, component ID, component name, component options, notify message มาจากอุปกรณ์โดยตรงหรือ NETPIE shadow data component metadata ถูกเก็บใน SQLite; สถานะควบคุมใช้เพื่อแสดงผลและ refresh ในแอป
ข้อมูลยืนยันตัวตนของอุปกรณ์ device password แบบ hash, NETPIE Client ID, NETPIE token/password มาจากผู้ใช้และ device profile ที่อุปกรณ์รายงาน เก็บใน SQLite เพื่อใช้ควบคุมอุปกรณ์ภายหลัง
ข้อมูล WiFi ระหว่าง setup SSID, WiFi password, security flag, signal strength มาจากอุปกรณ์และจากผู้ใช้ตอนตั้งค่า WiFi WiFi password ใช้ระหว่างส่งให้อุปกรณ์เพื่อเชื่อมต่อเครือข่าย และไม่ได้ถูกบันทึกลง SQLite โดย flow ปัจจุบัน

วิธีใช้ข้อมูล

  • ค้นหาอุปกรณ์ IoT ในเครือข่ายเดียวกันด้วย UDP discovery
  • ตั้งชื่อ จัดกลุ่ม และบันทึกอุปกรณ์ที่ผู้ใช้เลือกเพิ่มเข้ามาในแอป
  • ตั้งรหัสผ่านอุปกรณ์และตั้งค่า WiFi ให้กับอุปกรณ์ระหว่าง setup
  • อ่าน device profile, component metadata และสถานะของอุปกรณ์เพื่อแสดง UI ควบคุมที่เหมาะสม
  • ส่งคำสั่งควบคุม เช่น power, switch, button group, delay timer และ text field ไปยังอุปกรณ์
  • ตรวจสอบสถานะ online/offline ผ่าน NETPIE หรือผ่านการค้นหาอุปกรณ์ใน LAN

แอปส่ง packet ค้นหาอุปกรณ์ผ่าน UDP และส่งคำสั่งผ่าน TCP ไปยัง IP/port ของอุปกรณ์ ข้อมูลที่ส่งอาจรวม device password header, คำสั่ง setup, WiFi SSID, WiFi password, และคำสั่งควบคุม component

NETPIE

การจัดเก็บและระยะเวลาเก็บข้อมูล

ข้อมูลกลุ่ม อุปกรณ์ รหัสผ่านอุปกรณ์แบบ hash ข้อมูล MQTT/NETPIE และ component metadata ถูกเก็บไว้ในฐานข้อมูล SQLite ชื่อ app.db บนอุปกรณ์มือถือของผู้ใช้ ข้อมูลจะอยู่ในเครื่องจนกว่าผู้ใช้จะลบอุปกรณ์ ลบกลุ่ม ล้างข้อมูลแอป หรือถอนการติดตั้งแอป

แอปไม่ได้ระบุระบบสำรองข้อมูลหรือ sync ข้อมูลไปยัง backend ของผู้พัฒนาแอปในโค้ดปัจจุบัน แต่ระบบปฏิบัติการหรือบริการ backup ของอุปกรณ์ผู้ใช้อาจสำรองข้อมูลแอปตามการตั้งค่าของเครื่อง

สิทธิ์และการใช้เครือข่าย

แอปจำเป็นต้องใช้การเชื่อมต่อเครือข่ายเพื่อค้นหา ตั้งค่า และควบคุมอุปกรณ์ IoT

  • Internet / Network: ใช้สำหรับ NETPIE API, TCP direct control และ UDP discovery
  • Local network: ใช้ค้นหาและติดต่ออุปกรณ์ใน WiFi/LAN เดียวกัน
  • WiFi information: แอปใช้ IP address ของเครื่องเพื่อคำนวณ broadcast address สำหรับค้นหาอุปกรณ์

แอปไม่ได้ใช้ข้อมูลตำแหน่ง GPS จากโค้ดปัจจุบัน การแสดงรายการ WiFi ที่เลือกตอน setup มาจากอุปกรณ์ IoT ที่สแกนเครือข่ายและส่งผลลัพธ์กลับมาให้แอป

ทางเลือกและการควบคุมของผู้ใช้

  • ผู้ใช้สามารถเลือกไม่เพิ่มอุปกรณ์หรือไม่กรอกข้อมูล WiFi/NETPIE ได้ แต่ฟังก์ชัน setup/control บางส่วนอาจใช้งานไม่ได้
  • ผู้ใช้สามารถลบ Device Group หรืออุปกรณ์ที่เพิ่มไว้ได้จากภายในแอปตามหน้าจอที่รองรับ
  • ผู้ใช้สามารถล้างข้อมูลแอปจากการตั้งค่าระบบปฏิบัติการ หรือถอนการติดตั้งแอปเพื่อลบข้อมูล local database
  • หากเปลี่ยน token/password ของ NETPIE หรืออุปกรณ์ ผู้ใช้อาจต้อง setup อุปกรณ์ใหม่เพื่อให้ข้อมูลในแอปตรงกับค่าปัจจุบัน

ความปลอดภัย

แอป hash device password ก่อนบันทึกและใช้รหัสผ่านดังกล่าวในการยืนยันตัวตนกับอุปกรณ์ อย่างไรก็ตาม ข้อมูลบางส่วน เช่น NETPIE token/password และข้อมูลอุปกรณ์ ถูกเก็บไว้ในฐานข้อมูล local เพื่อให้แอปควบคุมอุปกรณ์ได้ ผู้ใช้ควรล็อกหน้าจอเครื่อง เก็บอุปกรณ์มือถือให้ปลอดภัย และหลีกเลี่ยงการติดตั้งแอปจากแหล่งที่ไม่น่าเชื่อถือ

ระหว่าง setup WiFi แอปส่ง SSID และ WiFi password ไปยังอุปกรณ์ผ่านเครือข่ายท้องถิ่น ผู้ใช้ควรทำขั้นตอน setup บนเครือข่ายที่เชื่อถือได้
